Basic Web Accessibility - Tabindex

Try and reach the element(s) below using just your keyboard. If you can't, use your mouse.

That was weird, wasn't it?

tabindex is an interesting attribute. It can be applied to any HTML element (but really shouldn't) in order to allow that element to be focusable and to be able
to participate in sequential keyboard navigation. tabindex comes in a few flavors per the MDN:
  • tabindex="0" - this element is focusable and reachable via sequential keyboard navigation, but its relative order is defined by DOM and whatever else is focusable around it.
  • tabindex="1", tabindex="2", tabindex="3", etc... - this element is focusable and reachable via sequential keyboard navigation, but its relative order is defined by the increasing order of tabindex values.
  • tabindex="-1" - this element is focusable but NOT reachable via sequential keyboard navigation. This is primarily used when you need to have a non-standard element be focusable for the purposes of Javascript.
There are a handful of HTML attributes which natively have a tabindex of 0. They include: button, a, input, option, select and textarea.
Generally speaking, this is all you should ever need. There are some circumstances where using tabindex="-1" or even tabindex="0" MIGHT be
warrented, but rarely, and there's pretty much never a good reason to use tabindex="positive integer". So if you find yourself doing it, stop and think.
Do you really want disrupt the actual flow of the DOM?
<button class="btn btn-primary" style="margin-bottom: 35px;" type="button" tabindex="1">A button</button> <button id="button" class="btn btn-primary" style="margin-bottom: 35px;" type="button" tabindex="5">CLICK ME</button> <button class="btn btn-primary" style="margin-bottom: 35px;" type="button" tabindex="2">Another button</button> <button class="btn btn-primary" style="margin-bottom: 35px;" type="button" tabindex="4">What is happening?</button> <button class="btn btn-primary" style="margin-bottom: 35px;" type="button" tabindex="3">Another button</button>
