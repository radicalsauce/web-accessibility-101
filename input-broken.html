No connection between label and input

Without an association between a label ("label for='someInput') and an input field ("id='someInput'), there is no way for
a screenreader to know which label goes with which input and it will simply read back, "edit text" without any other context.

Input field is not actually required

This input field has its label/input association correctly set up, but it doesn't actually tell the screenreader user that the field is required in order
to proceed. It, instead, relies solely on an asterisk to convey with information, which is not something a blind person is going to be able to see.
<label style="padding-right: 10px;"> Enter Name </label> <input id="input1" type="text"></input> <label for="input2" style="padding-right: 10px;"> Enter Email<span style="color:red;">*</span> </label> <input id="input2" type="text"></input>
